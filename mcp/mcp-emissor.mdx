## O que é o MCP Emissor?

O `thatsme-mcp-issuer` é um servidor MCP que conecta assistentes de IA, como Claude e Cursor, à sua conta emissora na That's Me. Com ele, você pode emitir certificados, consultar o funil de engajamento e exportar dados usando linguagem natural.

**Plano mínimo:** Pro. Quatro das cinco tools exigem plano Pro ou superior.

## Instalação (stdio local)

```bash
npx -y thatsme-mcp-issuer
```

É necessário configurar a variável de ambiente `THATSME_API_KEY`.

### Claude Desktop

Adicione ao seu `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "thatsme-issuer": {
      "command": "npx",
      "args": ["-y", "thatsme-mcp-issuer"],
      "env": {
        "THATSME_API_KEY": "sua_api_key_aqui"
      }
    }
  }
}
```

### Cursor

Adicione ao arquivo `.cursor/mcp.json`:

```json
{
  "mcpServers": {
    "thatsme-issuer": {
      "command": "npx",
      "args": ["-y", "thatsme-mcp-issuer"],
      "env": {
        "THATSME_API_KEY": "sua_api_key_aqui"
      }
    }
  }
}
```

## Conexão remota (HTTP/SSE)

Para clientes que suportam MCP remoto:

```text
URL: https://mcp.thatsme.com.br/mcp
Transport: Streamable HTTP
```

Envie o header `Authorization: Bearer SUA_API_KEY` em cada requisição.

## Tools disponíveis

### 1. `search_events`

Busca eventos criados pela sua organização.

| Parâmetro | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `query` | string | Sim | Nome do evento |
| `limit` | number | Não | Máximo de resultados (padrão: 10, máximo: 50) |

**Exemplo de prompt:**
> "Liste os eventos da minha organização que contêm 'DevFest'"

**Plano mínimo:** Starter

---

### 2. `issue_certificates`

Emite certificados digitais para uma lista de destinatários.

| Parâmetro | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `event_id` | string | Sim | ID do evento |
| `badge_id` | string | Sim | ID do badge/TM |
| `recipients` | array | Sim | Lista de destinatários (máximo: 500) |
| `recipients[].email` | string | Sim | E-mail do destinatário |
| `recipients[].name` | string | Sim | Nome completo |
| `recipients[].reason` | string | Não | Motivo da certificação |

**Exemplo de prompt:**
> "Emita certificados do DevFest 2025 para os 3 palestrantes: joao@email.com (João Silva), maria@email.com (Maria Santos), pedro@email.com (Pedro Lima)"

**Plano mínimo:** Pro

---

### 3. `get_engagement_funnel`

Retorna o funil de engajamento de um evento: emitidos → aceitos → visualizados → compartilhados.

| Parâmetro | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `event_id` | string | Sim | ID do evento |

**Exemplo de prompt:**
> "Mostre o funil de engajamento do evento DevFest 2025"

**Resposta inclui:** contagens e taxas calculadas (`acceptance_rate`, `view_rate`, `share_rate`).

**Plano mínimo:** Pro

---

### 4. `list_recipients`

Lista os destinatários de um evento com o status do certificado: pendente, aceito ou recusado.

| Parâmetro | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `event_id` | string | Sim | ID do evento |
| `page` | number | Não | Página (padrão: 1) |
| `status` | enum | Não | Filtro: `all`, `accepted`, `pending`, `refused` |

**Exemplo de prompt:**
> "Quem ainda não aceitou o certificado do evento DevFest 2025?"

**Nota:** Os e-mails são parcialmente mascarados por privacidade, por exemplo: `jo***@email.com`.

**Plano mínimo:** Pro

---

### 5. `export_csv`

Exporta dados de emissão em formato CSV.

| Parâmetro | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `period` | enum | Não | `30d`, `90d`, `all` (padrão: `30d`) |

**Exemplo de prompt:**
> "Exporte os dados de emissão dos últimos 90 dias em CSV"

**Plano mínimo:** Pro

2. Clique em **Nova chave de API**
3. Copie a chave, pois ela não será exibida novamente
