---
api: "POST /api/v1.0/events"
title: "Criar Evento"
---

Cria um novo evento para o **emissor autenticado**.

<Request method="POST" url="/api/v1.0/events" auth="bearer" />

<Note>
  Após criar o evento, utilize `GET /api/v1.0/events` para localizar o `event_id` (ou capture o `event_id` da resposta) e iniciar emissões.
</Note>

---

## Parâmetros do corpo (body)

<ParamField body="category" type="string" required>
  Categoria do evento. Valores aceitos: `SPORTS`, `EDUCATION`, `CORPORATE`, `SOCIOCULTURAL`.
</ParamField>

<ParamField body="name" type="string" required>
  Nome do evento.
</ParamField>

<ParamField body="description" type="string" required>
  Descrição do evento.
</ParamField>

<ParamField body="location_type" type="string" required>
  Tipo de localização do evento. Valores aceitos: `online` ou `physical`.

  - `online`: `city` e `state` são definidos como `ONLINE`
  - `physical`: `city` e `state` tornam-se obrigatórios
</ParamField>

<ParamField body="state" type="string">
  Estado (UF) do evento (obrigatório quando `location_type` for `physical`).

  Valores aceitos: `AC`, `AL`, `AP`, `AM`, `BA`, `CE`, `DF`, `ES`, `GO`, `MA`, `MT`, `MS`, `MG`, `PA`, `PB`, `PR`, `PE`, `PI`, `RJ`, `RN`, `RS`, `RO`, `RR`, `SC`, `SP`, `SE`, `TO`.
</ParamField>

<ParamField body="city" type="string">
  Cidade do evento (obrigatório quando `location_type` for `physical`).
</ParamField>

<ParamField body="initial_date" type="string">
  Data e hora de início (ISO 8601).
</ParamField>

<ParamField body="finish_date" type="string">
  Data e hora de término (ISO 8601).
</ParamField>

<ParamField body="official_url" type="string">
  URL oficial do evento.
</ParamField>

<ParamField body="image_url" type="string">
  URL da imagem do evento.
</ParamField>

<ParamField body="image_key" type="string">
  Chave da imagem do evento (S3).
</ParamField>

<ParamField body="is_private" type="boolean">
  Define se o evento é privado.
</ParamField>

<ParamField body="date_mode" type="string">
  Modo de datas do evento. Valores aceitos: `single_date`, `date_range`, `recurring`, `always_available`.

  Regras:

  - `always_available`: datas não são obrigatórias
  - `single_date` e `date_range`: `initial_date` é obrigatório (e `finish_date` quando aplicável)
  - `recurring`: exige `recurrence_type` e `recurrence_start_date`
</ParamField>

<ParamField body="start_time" type="string">
  Horário de início do evento (quando aplicável).
</ParamField>

<ParamField body="end_time" type="string">
  Horário de término do evento (quando aplicável).
</ParamField>

<ParamField body="recurrence_type" type="string">
  Tipo de recorrência do evento (quando `date_mode` for `recurring`).\
  Valores aceitos: `WEEKLY`, `BIWEEKLY`, `MONTHLY`, `BIMONTHLY`, `QUARTERLY`, `SEMIANNUAL`, `ANNUAL`.
</ParamField>

<ParamField body="recurrence_start_date" type="string">
  Data de início da recorrência (ISO 8601) (quando `date_mode` for `recurring`).
</ParamField>

<Info>
  Obrigatoriedade de datas:

  - Para eventos com data (`single_date`, `date_range`, `recurring`), a data é obrigatória.
  - Para eventos `always_available` (“Sempre disponível”), as datas podem ser omitidas.
</Info>

<ParamField body="connected_event_ids" type="array">
  Lista de IDs de eventos conectados.
</ParamField>

<Note>
  Eventos conectados podem ser utilizados pelo motor de recomendações para sugerir eventos relacionados após a emissão de certificados.
</Note>

<Info>
  Campos e regras condicionais (ex.: obrigatoriedade de `city`/`state`) devem ser respeitados para evitar erro de validação.
</Info>

---

</ResponseField>

</ResponseField>

</ResponseField>

<ResponseField name="initial_date" type="string">

</ResponseField>

<ResponseField name="finish_date" type="string">

</ResponseField>

<ResponseField name="image_url" type="string">
