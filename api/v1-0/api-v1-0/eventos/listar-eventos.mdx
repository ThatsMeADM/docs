---
api: "GET /api/v1.0/events"
title: "Listar Eventos"
---

Retorna uma lista **enxuta** de eventos do **próprio emissor autenticado**, com foco em localizar o `event_id` necessário para iniciar emissões de certificados.

Este endpoint exige autenticação via **Bearer Token** e **não** lista eventos de outros emissores.

<Request method="GET" url="/api/v1.0/events" auth="bearer" />

<Note>
  Uso recomendado: utilize este endpoint para localizar o `event_id`\
  Para obter informações completas de um evento, utilize`GET /api/v1.0/events/{id}`
</Note>

---

## Parâmetros de consulta

<ParamField query="limit" type="number">
  Quantidade máxima de eventos retornados por página.
</ParamField>

<ParamField query="offset" type="number">
  Posição inicial da listagem para paginação.
</ParamField>

<ParamField query="q" type="string">
  Busca textual por nome do evento (case-insensitive). Recomendado para localizar rapidamente o evento desejado.
</ParamField>

<ParamField query="status" type="string">
  Filtra por status do evento (ex.: `ACTIVE` or `COMPLETED`).
</ParamField>

<ParamField query="category" type="string">
  Filtra por categoria do evento (ex.: `EDUCATION`, `SPORTS`, `CORPORATE`, `SOCIOCULTURAL`).
</ParamField>

<ParamField query="created_from" type="string">
  Filtra eventos criados a partir de uma data (ISO 8601).
</ParamField>

<ParamField query="created_to" type="string">
  Filtra eventos criados até uma data (ISO 8601).
</ParamField>

<ParamField query="sort" type="string">
  Campo de ordenação. Padrão recomendado: `created_at`.
</ParamField>

<ParamField query="order" type="string">
  Ordem de ordenação: `asc` ou `desc`.\
  Padrão recomendado: `desc`.
</ParamField>

<Info>
  `limit` controla quantos eventos retornam por chamada.\
  `offset` controla a partir de qual posição a listagem começa (paginação).

  Para localizar eventos rapidamente, recomenda-se combinar `q` + `status` e ordenar por `created_at`.
</Info>

---

## Resposta

<ResponseField name="data" type="array">
  Lista de eventos do emissor autenticado, retornando somente campos essenciais para identificação.

  <Expandable title="items">
    <ResponseField name="event_id" type="string">
      Identificador único do evento
    </ResponseField>
    <ResponseField name="name" type="string">
      Nome do evento
    </ResponseField>
    <ResponseField name="status" type="string">
      Status atual do evento (ex.: ACTIVE, COMPLETED)
    </ResponseField>
    <ResponseField name="category" type="string">
      Categoria do evento
    </ResponseField>
    <ResponseField name="created_at" type="string">
      Data de criação do evento (ISO 8601)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="number">
  Quantidade total de eventos encontrados para os filtros aplicados
</ResponseField>

<ResponseField name="limit" type="number">
  Limite aplicado na consulta
</ResponseField>

<ResponseField name="offset" type="number">
  Offset aplicado na consulta
</ResponseField>

## Exemplos de uso

<AccordionGroup>
  <Accordion title="Listar eventos mais recentes">
    ```http
    GET /api/v1.0/events?limit=10&offset=0&sort=created_at&order=desc
    ```
  </Accordion>
  <Accordion title="Buscar evento por nome (para obter o event_id)">
    ```http
    GET /api/v1.0/events?q=workshop&limit=10&offset=0&sort=created_at&order=desc
    ```
  </Accordion>
  <Accordion title="Filtrar somente eventos ativos">
    ```http
    GET /api/v1.0/events?status=ACTIVE&limit=10&offset=0&sort=created_at&order=desc
    ```
